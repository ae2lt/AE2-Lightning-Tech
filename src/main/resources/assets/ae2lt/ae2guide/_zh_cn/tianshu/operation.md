# 端口、任务与快速计算

## 天枢超算端口

结构成形后，<ItemLink id="ae2lt:tianshu_supercomputer_port" /> 会将整套多方块作为名为“天枢超算中心”的合成 CPU 发布到 ME 网络。端口仅在结构成形后允许六个方向连接 ME 线缆；未成形时无法接线。待机功耗为 **8 AE/t**，整套结构占用 **1 个频道**。

端口离线、ME 网络断电或结构解体时，天枢超算中心不会接受新任务。已经保存的任务状态不会因区块卸载而丢失；结构恢复并重新联网后可以继续执行。

天枢超算不会强制加载其覆盖的区块。任一结构区块未加载时，端口会断开并暂停任务；完整结构重新加载后会自动复扫和续接。跨区块建造时，区块加载器应覆盖整座结构。

## 天枢样板编码终端

<ItemLink id="ae2lt:tianshu_pattern_encoding_terminal" /> 在普通样板编码终端的基础上提供闭环编写、样板上传与库存维持配置。菜单会按稳定顺序绑定第一座可用的已成形天枢；如果打开时尚无目标，则可在第一座天枢成形上线时完成初始绑定。本次菜单会话一旦绑定机器 UUID，就不会切换或回退到其他机器；已绑定机器离线时相关写操作会直接失败。

当前绑定天枢的闭环样板仓会作为额外样板库存显示在该终端的网络内容列表中。只有天枢 CPU 结构已成形且端口在线时才会显示，结构失效后条目立即隐藏。玩家可以从列表取出闭环样板进行检查或修改，重新放回样板仓仍需使用终端的上传功能。

库存维持总览会单独显示已配置但当前库存为零的物品。按住 Shift 点击条目即可编辑对应的库存维持规则。若迁移后的旧数据超过当前硬上限，总览会明确标记有界恢复页；逐条删除可见项后，后续持久化条目会在新 revision 中继续显示。

## 共享容量与多任务

天枢超算中心与普通合成 CPU 的主要差异是可以同时保存并执行多个合成任务。

* 每个任务开始时，会从总合成容量中预留该计划所需的字节
* 只要剩余容量足够，新的任务可以继续分配给同一座天枢超算中心
* 任务完成或取消后，其预留容量会释放
* 所有活动任务共享核心配置的成功派发预算；每个任务分别拥有完整的每 tick 最大复制预算，但只有兼容批量执行的样板与执行端能够使用超过单份派发的部分

如果合成确认界面提示 CPU 容量不足，应等待现有任务释放容量，或增加存储超算单元、改用更高等级的主超算单元。

## 控制器界面

右击 <ItemLink id="ae2lt:tianshu_supercomputer_controller" /> 可以查看：

* 结构是否成形，以及未成形时优先检测到的问题
* 当前主超算单元等级
* 存储、并行与增幅超算单元数量
* 总合成容量、每 tick 成功派发数、每 tick 最大复制数及是否达到派发上限；AE2 的 CPU 选择列表只按成功派发能力显示并行数
* 自动搭建外壳与快速计算开关

更换核心舱单元会使结构暂时解体。若端口仍保存活动任务，新配置会在全部既有任务结束后生效；恢复原结构与网络连接可使任务继续执行。

## 快速计算

“快速计算”默认开启，用于缩短大型合成计划的计算时间。它只影响提交任务前的计划计算，不改变配方、原料需求、合成容量或任务执行速度。

快速计算无法处理部分计划或发生异常时，会自动回退到 AE2 的标准计算方式。若需要排查样板兼容性或对比计算结果，可以在控制器中将其关闭。

快速计算的启用条件按整个 ME 网络汇总。同一网络中只要仍有其他支持快速计算的活动合成 CPU 允许该功能，网络仍可能使用快速计算；需要完全停用时，应关闭其对应设置或使这些 CPU 离线。

## 故障排查

* **无法成形：** 检查固定外壳、玻璃与冷却单元，确认端口恰好一个、主超算单元位于中心，并按主核心等级填满 26 个外围槽；有限等级至少需要一个并行单元，多维核心不能搭配存储、并行或增幅单元
* **结构方块齐全但仍未成形：** 清除展示结构中省略的必要空气区域；其中不能放置线缆、照明或装饰方块
* **自动搭建无反应：** 根据控制器提示清除阻挡，并确认所有材料位于玩家背包
* **合成确认界面中没有天枢超算中心：** 检查端口的线缆连接、ME 网络供电与结构成形状态
* **跨区块加载后端口离线：** 确认区块加载器覆盖整座天枢超算，等待自动复扫
* **无法接受新任务：** 检查剩余合成容量；总容量可能已经由其他活动任务预留
* **成功派发较高但加工速度未提高：** 检查样板供应器与加工设备数量、加工设备速度、原料供应以及网络传输能力
* **最大复制较高但普通处理样板仍按单份发送：** 普通 AE2 加工设备使用原生单份派发路径；批量复制只适用于兼容的样板、供应器与执行端
