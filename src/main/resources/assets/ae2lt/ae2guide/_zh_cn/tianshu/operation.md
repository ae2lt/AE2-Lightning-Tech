# 端口、任务与快速计算

## 天枢超算阵列端口

结构成形后，<ItemLink id="ae2lt:tianshu_supercomputer_port" /> 会将整套多方块作为名为“天枢超算阵列”的合成 CPU 发布到 ME 网络。端口仅在结构成形后允许六个方向连接 ME 线缆；未成形时无法接线。待机功耗为 **8 AE/t**，整套结构占用 **1 个频道**。

端口离线、ME 网络断电或结构解体时，天枢超算阵列不会接受新任务。已经保存的任务状态不会因区块卸载而丢失；结构恢复并重新联网后可以继续执行。

天枢超算阵列不会强制加载其覆盖的区块。任一结构区块未加载时，端口会断开并暂停任务；完整结构重新加载后会自动复扫和续接。跨区块建造时，区块加载器应覆盖整座结构。

## 天枢样板编码终端

[天枢样板编码终端](pattern-encoding-terminal.md)在普通样板编码终端的基础上提供处理样板增强编码、样板上传、闭环样板编写与库存维持配置，是编写和管理天枢专用样板的入口，详见其专属页面。

## 共享容量与多任务

天枢超算阵列与普通合成 CPU 的主要差异是可以同时保存并执行多个合成任务。

* 每个任务开始时，会从总合成容量中预留该计划所需的字节
* 只要剩余容量足够，新的任务可以继续分配给同一座天枢超算阵列
* 任务完成或取消后，其预留容量会释放
* 所有活动任务共享核心配置的成功派发预算；每个任务分别拥有完整的每 tick 最大复制预算，但只有兼容批量执行的样板与执行端能够使用超过单份派发的部分

如果合成确认界面提示 CPU 容量不足，应等待现有任务释放容量，或增加天枢存储单元、改用更高等级的主核心。

## 控制器界面

右击 <ItemLink id="ae2lt:tianshu_supercomputer_controller" /> 可以查看：

* 结构是否成形，以及未成形时优先检测到的问题
* 当前天枢主核心等级
* 并行、增幅与存储单元数量，以及闭环样板仓、闭环种子存储器数量
* `R`、发配增益 `Gd`、容量增益 `Gs`，以及原始发配和有效成功派发预算 `D`
* 内置容量、增幅后的外部容量、总合成容量、每个虚拟 CPU 的每 tick 最大复制数 `T` 及是否达到派发上限；AE2 的 CPU 选择列表只按成功派发能力显示并行数
* 自动搭建外壳与快速计算开关

更换核心舱单元，或拆装冷却兼容位上的闭环存储，会使结构在方块缺失期间暂时解体。若端口保存活动任务，新配置会在全部既有任务结束后生效；恢复原结构与网络连接可使任务继续执行。相变冷却单元可以占用同一冷却兼容位；这种配置结构有效，但不提供对应的闭环容量。

## 快速计算

“快速计算”默认开启，用于缩短大型合成计划的计算时间。它只影响提交任务前的计划计算，不改变配方、原料需求、合成容量或任务执行速度。

快速计算无法处理部分计划或发生异常时，会自动回退到 AE2 的标准计算方式。若需要排查样板兼容性或对比计算结果，可以在控制器中将其关闭。

快速计算的启用条件按整个 ME 网络汇总。同一网络中只要仍有其他支持快速计算的活动合成 CPU 允许该功能，网络仍可能使用快速计算；需要完全停用时，应关闭其对应设置或使这些 CPU 离线。

## 故障排查

结构未成形时，控制器界面会直接显示优先检测到的问题，应先按该提示处理。

**结构与搭建**

* **无法成形：** 对照[手动搭建规则](construction.md#手动搭建规则)逐项检查——外壳与玻璃齐全、端口恰好一个、17 个非端口冷却兼容位填满相变冷却单元或闭环存储、天枢主核心位于核心舱中心、26 个外围槽用当前主核心支持的单元填满
* **结构方块齐全但仍未成形：** 清除展示结构中省略的必要空气区域；其中不能放置线缆、照明或装饰方块
* **自动搭建无反应：** 根据控制器提示清除阻挡，并确认所有材料位于玩家背包

**联网与任务**

* **合成确认界面中没有天枢超算阵列：** 检查端口的线缆连接、ME 网络供电与结构成形状态
* **跨区块加载后端口离线：** 确认区块加载器覆盖整座天枢超算阵列，等待自动复扫
* **无法接受新任务：** 剩余合成容量可能已被其他活动任务预留；等待任务结束，或增加存储单元

**性能未达预期**

* **成功派发较高但加工速度未提高：** 瓶颈通常在执行侧——检查样板供应器与加工设备数量、设备速度、原料供应以及网络传输能力
* **最大复制较高但普通处理样板仍按单份发送：** 普通 AE2 加工设备每次调用只接受一份样板；批量复制只适用于兼容的样板、供应器与执行端，详见[批量复制](computing-units.md#批量复制)
